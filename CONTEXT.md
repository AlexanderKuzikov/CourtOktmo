# CONTEXT.md — CourtOktmo

## Назначение

**CourtOktmo** — определение ОКТМО/ОКАТО/ОКПО для адресов судов РФ.
Основной источник — **ГАР (ФИАС)**, резервный — **DaData API**.

Является третьим компонентом экосистемы сбора данных о судах, после:
- CourtHarvest2 (DaData API: code, name, address, ИНН)
- CourtSudrf (sudrf.ru: ПСП, телефоны, email)

---

## Этапы сбора ОКТМО

### 1. Первичный сбор — из ГАР (однократно, со свежим архивом)

Пайплайн `gar/`:
```
gar/gar_xml.zip (54 ГБ) → extract → D:\gar_extracted\
  → build-addr-map → addr-map-{region}.ndjson
  → build-mun-map  → mun-map-{region}.ndjson  (содержит ОКТМО)
  → build-houses-map → houses-map-{region}.ndjson
  → build-street-houses-map → street-houses-map-{region}.ndjson
  → resolve-court-address → resolved-{region}.json
```

Результат: **10 208 записей, 8 793 с ОКТМО (86.1%)** в `data/resolved-all.json`.

### 2. Поддержание актуальности — через DaData

При каждом harvest в CH2 выявляются изменившиеся адреса.
Только для них — запрос ОКТМО через DaData suggestAddress.
ГАР для инкремента не перезапускаем.

## Формат ПСП (суффиксный)

Утверждён в CONTEXT.md CourtSudrf раздел 11.
Плоский JSON без вложенных массивов:

```json
{
  "code": "43RS0018",
  "address": "612600, Кировская обл., г. Котельнич, ул. Советская, д. 27",
  "phone": "(83342) 4-07-57",
  "okmo": "33614000",
  "psp_count": 2,
  "psp_address_0": "612140, пгт. Даровской, ул. М. Гвардия, д. 21 А",
  "psp_phone_0": "(83336) 2-11-06",
  "psp_okmo_0": "33614404",
  "psp_address_1": "612270, г. Орлов, ул. Кирова, д. 65",
  "psp_phone_1": "(83365) 2-86-27",
  "psp_okmo_1": "33615404"
}
```

Правила:
- `psp_count = 0` — ПСП нет, суффиксные поля отсутствуют
- `psp_count = N` — поля с суффиксами `_0` … `_N-1`
- Индексация строго от 0 до N-1, без пропусков
- Буфер: 0..9 (макс 10 ПСП на 1 суд)
- При добавлении/удалении ПСП — переиндексация всех записей

## Формат временного файла ПСП

`data/psp_addresses.json` — 155 ПСП-адресов из CourtSudrf.
```json
{
  "code": "59RS0006",      // оригинальный код суда
  "psp_index": "59PSP0001", // внутренний идентификатор ПСП
  "address": "614026, г. Пермь, ул. Валёжная, д. 13",
  "name": "Орджоникидзевский районный суд г. Перми (ПСП)"
}
```

⚠️ Код суда (`code`) — всегда оригинальный код RRTTNNNN.
Искусственные коды (PSP0001) — только во вспомогательных полях.

## Ключевые архитектурные решения

1. **ОКТМО — только из ГАР.** DaData — только на этапе поддержания (изменился адрес → DaData suggestAddress).
2. **DaData suggestParty** — только для ОКПО (229 судов с ИНН), не для ОКТМО.
3. **Исходные коды судов (RRTTNNNN) — константа.** Никаких производных кодов.
4. **ПСП — суффиксные поля на корне.** Не отдельные записи.

## Источники ОКТМО

### 1. ГАР (ФИАС) — основной источник

Официальный реестр Росреестра. XML-файлы выгружаются ежемесячно (~54 ГБ).

| Файл ГАР | Что даёт |
|----------|----------|
| `AS_ADDR_OBJ` | Адресные объекты (регионы, города, улицы) с OBJECTID/OBJECTGUID |
| `AS_HOUSES` | Дома с OBJECTGUID и номерами |
| `AS_MUN_HIERARCHY` | Муниципальная иерархия с **ОКТМО** в PATH |

**Пайплайн:** build-addr-map → build-mun-map → build-houses-map → build-street-houses-map → resolve-court-address

### 2. DaData /suggest/party (по ИНН) — для судов с ИНН

Для 229 судов (AS, OS, AA, AO, AJ, KJ, AV, KV, OV, VS). Возвращает: **okpo, okato, oktmo**.

### 3. DaData /suggest/address — фоллбэк

Для адресов, не найденных в ГАР, и для детекции изменений (при переездах).

### Сравнение методов

| Характеристика | suggest/party | suggest/address |
|---------------|:-------------:|:---------------:|
| ОКТМО | ✅ | ✅ |
| ОКАТО | ✅ | ✅ |
| ОКПО | ✅ | ❌ |
| Нужен ИНН | ✅ | ❌ |
| Нужен адрес | ❌ | ✅ |
| Запросов | 229 | ~10 000 + ~155 ПСП |

## Алгоритм сбора

1. **party** — все суды с ИНН из CourtHarvest2 (229 запросов)
2. **address** — main-адреса всех судов из CourtHarvest2 (10 000 запросов)
3. **address (PSP)** — ПСП-адреса из CourtSudrf (155 запросов)

## Формат данных

Совместим с CourtHarvest2 и CourtSudrf:

```json
{
  "code": "59RS0001",
  "name": "Дзержинский районный суд г. Перми",
  "inn": null,
  "court_type": "RS",
  "address": "614068, Пермский край, г. Пермь, ул. Плеханова, д. 40",
  "okmo": "57701000001",
  "okato": "57401365000",
  "okpo": null
}
```

Поля ОКТМО/ОКАТО/ОКПО добавляются к существующим записям без удаления других полей.

## Технические решения

### Ключи DaData

Переиспользуются ключи из CourtHarvest2. 4 ключа по ~9 500 запросов.

### Rate limiting

minTime = 200ms (5 запросов/с). С 4 ключами — 20 запросов/с.

### ESM

Проект на ESM (`"type": "module"`). Все импорты с расширением `.js`.

### TypeScript 7 + erasableSyntaxOnly

Те же правила, что и в CourtHarvest2:
- Без `constructor(private x: T)` — явное поле + присваивание
- Без декораторов и enum
- strict mode, noUnusedLocals, noUnusedParameters

## Архитектура поиска ОКТМО (актуальная)

### Основной принцип

ОКТМО определяется **населённым пунктом**, а не улицей и домом.
Проверено на ГАР (регион 01 — Адыгея): все объекты внутри одного НП имеют единый ОКТМО.
Исключения — СНТ/ДНТ (приписаны к другому муниципалитету), для судов неактуально.

### Алгоритм (принято)

```
адрес → населённый пункт → OBJECTID locality → mun-map → ОКТМО
```

Не требует номера дома и улицы. Решает:
- P1 (8% адресов с нестандартными форматами: влд, соор, литеры, двойные адреса)
- P3 (ПСП-адреса — 144 из 146)

### Формат входных адресов

Принят **набор правил (мягкая нормализация)**, а не жёсткий шаблон.
Адреса хранятся в исходном виде (как пришли из CH2/CSRF).
Нормализация выполняется «на лету» при поиске ОКТМО.

### Москва и Санкт-Петербург

Для Москвы (132 внутригородских МО) и СПб (111 МО) требуется уточнение:
- через ГАР: улица → parent OBJECTID → внутригородское МО → ОКТМО (предпочтительно)
- через DaData suggestAddress как фоллбэк (запросов немного)

## Отложенные решения

### Динамический словарь именованных сущностей (идея)

Словарь сырых названий из CH2 + CSRF → эталон ГАР.
Наполняется от реальных адресов судов (~20 000 записей), негенеральный.
Формат: `{raw, type, objectid, oktmo, canon}`.
Хранить: `CourtOktmo/data/dictionary.json`.
Наполнение: однократный проход по всем адресам + инкрементальное пополнение.
Отложен до завершения текущего этапа P1.

## Ограничения

1. **DaData suggest/address может не найти адрес** — если адрес указан в нестандартном формате (сокращения, старые названия). Такие адреса пропускаются (около 5%).
2. **Мировые судьи (MS)** — в CourtSudrf нет адресов MS, поэтому для них адреса берутся из CourtHarvest2.
3. **Лицензия DaData** — бесплатный ключ: 10 000 запросов/день на suggest + party + address. При полном сборе (10 000+) требуется 2 ключа или 2 дня.
4. **Новые территории (ДНР, ЛНР, Запорожская, Херсонская обл)** — адреса отсутствуют во всех источниках (93 суда). Отложено.
